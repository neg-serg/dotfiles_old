 1. INTRO                                      *colorv-intro*

*ColorV* is a color view/pick/edit/design/scheme tool in vim.

It makes handling colors much easier.  

With it, you can:

View colors Easier~
    ':ColorV' (<leader>cv): show ColorV window

    ':ColorVView' (<leader>cw): show ColorV window with color text under cursor.

    ':ColorVPreview' (<leader>cpp): Preview colors in current buffer

Edit colors Easier~
    ':ColorVEdit' (<leader>ce): Edit color text under cursor

    ':ColorVEditAll' (<leader>cE): Edit color text under cursor and change all in current buffer.

    ':ColorVInsert' (<leader>cii): Insert color with ColorV window.

Design Colors Easier~
    ':ColorVName' (<leader>cn): show color name list window.

    ':ColorVList Hue' (<leader>cgh) generate Hue list with color text under cursor.

    ':ColorVTurn2 {hex1} {hex2}' (<leader>cgg) generate Hue list from hex1 to hex2.

    ':ColorVPicker' (<leader>cd): show a GUI color picker.

Design Schemes Easier~
    ':ColorVScheme' (<leader>css) Fetch scheme from Kuler or ColourLover

    ':ColorVSchemeFav' (<leader>csf) Show Faved schemes

    ':ColorVSchemeNew' (<leader>csn) Create a new scheme

And More~
    You can even use it under 8/256 color terminal.

Get latest and Post issues at https://github.com/Rykka/colorv.vim

 2. INSTALL                                    *colorv-install*

1.Using [Vundle.vim] https://github.com/gmarik/vundle 

  Add this Line to your vimrc::

    Bundle 'Rykka/colorv.vim' 
    " needed for fetching schemes online.
    Bundle 'mattn/webapi-vim'
 
2.Using [Vim.org] http://www.vim.org/scripts/script.php?script_id=3597

    Extract to your ~/.vim folder

    NOTE   Get Latest version
           https://github.com/Rykka/colorv.vim/

    NOTE   webapi.vim_ is needed for fetching schemes online.

.. _webapi.vim: https://github.com/mattn/webapi-vim

 3. QUICKSTART                                 *colorv-quickstart*

                                                            *colorv-window*
    ColorV Window~
        |:ColorV| (<leader>cv)

    View and Edit color in window.~
        Double Click/ <Space> / <Enter>
        +/-(Wheel Up/Down) on items.

    Copy and Paste~
        cc/yy
        p/<Ctrl-V>

                                                            *colorv-names*
    Colornames list (W3C).~
        |:ColorVName| (<leader>cn)

                                                            *colorv-view*
    View color-text.~
        |:ColorVView| (<leader>cw)

    NOTE:   Color-texts are words like:
            FF00CC #ff9744 rgb(33,44,155) OrangeRed cadetblue

                                                            *colorv-edit*
    Insert color-text.~
         |:ColorVInsert| (<leader>cii)
    Edit color-text.~
         |:ColorVEdit| (<leader>ce)
                                                            
                                                            *colorv--generate*
    Generate a list of color of color-text.~
        e.g: <leader>cg1 or ':ColorVList Hue 20 5'

        see |colorv-colors-generate| for details

    Generate a list from color1 to color2.~
        e.g: ':ColorVTurn2 ff0000 334455'

                                                            *colorv--schemes*
    A set of colors.~
        e.g: <leader>css or ':ColorVScheme' to show a random scheme.
             <leader>csf or ':ColorVSchemeFav' to show a Faved schemes.
             <leader>csn or ':ColorVSchemeNew' to Create new schemes.
        press 'f' in scheme window to fav a scheme, and 'F' in the Fav window to unfav it.

        if there are multi schemes , press 'n' and 'p' to navigate

    NOTE: webapi.vim is needed to fetch schemes from Kuler or colourlovers

                                                            *colorv-preview*
    Preview all color-text in current file~
        <leader>cpp or |:ColorVPreview|

                                                            *colorv-picker*
    A GUI colorpicker.~
        <leader>cd or |:ColorVPicker|

 3.1 TROUBLESHOOTING                           *colorv-troubleshooting*
    
    1. Too Slow~
    NOTE   If your vim is not compiled with 'python' or 'python3'
           Then it's 3~10 times slower when previewing files.

    2. ColorPicker~

        For windows users:
        You should have python and pygtk installed. 
        and have 'python2' in your $PATH.

        OR install gtk package and compile it with autoload/colorv/Makefile

        And, gtk is buggy in windows.


    3. Solve Bugs~
    NOTE   Get the latest ColorV version.
           Get the latest vim version.
           Check if your vim compiled with +python or +python3.
           If have errors with python. 
           ':let g:colorv_has_python=0' to stop using python.

    4.Report bugs~
        You can report bugs at https://github.com/Rykka/ColorV/issues .

 4. COMMANDS                                   *colorv-commands*

                                                    *:ColorV*
    |:ColorV|       ColorV window in previous size.
    |:ColorVMid|    |colorv-mid| window.
    |:ColorVMin|    |colorv-min| window.
    |:ColorVMax|    |colorv-max| window.
    
                                                    *:ColorVName*
    |:ColorVName|     Color-Name list window

                                                    *:ColorVList*
    |:ColorVList| {type} {nums} {step}
                    Generate color list with color-text under curos.
                    See details of {args} in |colorv-color-text|

                                                    *:ColorVTurn2*
    |:ColorVTurn2| {hex1} {hex2}
                    Generate color list from hex1 to hex2.

                                                    *:ColorVView*
    |:ColorVView|     Open the ColorV window with word under cursor.
                    the word could be in |colorv-color-text| .



                                                    *:ColorVEdit*
    |:ColorVEdit|     Open the ColorV window with word under cursor. And
will change the color-text after closing the ColorV window.

                                                    *:ColorVEditAll*
    |:ColorVEditAll|  Open the ColorV window with word under cursor. And
will change all color-text in current buf after closing the ColorV window.
    
                                                    *:ColorVEditTo*
    |:ColorVEditTo| {fmt}
                      Open the ColorV window with word under cursor. And
                      Change to the format later.
                      {fmt}: (NAME\HEX\RGB\0x\RGBA...) 
                      See details of {fmt} in |colorv-color-text|

                                                    *:ColorVInsert*
    |:ColorVInsert| {fmt}  Open the ColorV window. And
will insert the color-text after closing the ColorV window.
                      fmt can be empty.
                      See details of {fmt} in |colorv-color-text|

                                                    *:ColorVPicker*
    |:ColorVPicker|     Open a gtk colorpicker to pick colors.
                      default use Python2 version.
                      If you have error with python version.
                      it will try C version. 
                      else it will print out the error.

                      Python2 version colorpicker
                      requires |g:colorv_python_cmd| in $PATH and
                      python installed with pygtk-2.0 package.
                      
                      C version colorpicker needs  |:make|
                      with makefile in 'autoload/colorv/Makefile'
                      gtk+-2.0 lib is needed.


                                                    *:ColoVPreview*
    |:ColorVPreview|    highlight color texts in current buffer.
                      invalid color text will not be highlighted.
                      e.g: rgb(256,256,256) , which exceed range of 0~255


                                                    *:ColorVScheme*
    |:ColorVScheme|      Show a random scheme, or search a scheme of
provided hex.
    also 'top' ,'new' can be used to get scheme list
                      f to fav one scheme
                                                    *:ColorVSchemeNew*
    |:ColorVSchemeNew| {num}   Create a new scheme.
                      e to edit the color
                      {num} is the color numbers
                      when omitted, will create 5 color scheme.

                    
                                                    *:ColorVSchemeFav*
    |:ColorVSchemeFav|   Show a Scheme Fav Window.
                       F to unfav one scheme

                                                    *:ColorVClear*
    |:ColorVClear|      clear all the previewing colors.

                                                    *:ColorVquit*
    |:ColorVQuit|       Quit ColorV window.

 5. MAPPINGS                                   *colorv-mappings*
    
    Global Mappins.~
    
    All global mappings start with '<leader>c' , 
    You can change it with |g:colorv_global_leader|
    And you can redefine each mapping with each command, 
    then the default one will not be defined.
    
    If you don't want any global mappings at all.
    set |g:colorv_no_global_map| to 1

    NOTE  If other plugins have alredy taken the mapping. 
          It will not exist. You should redfine it In your vimrc.

                                               *colorv-mapping-buffer*

    Mappings in ColorV window buffer~
    
    Size~
    z/Z           toggle ColorV window size (max/mid/min)

    M            add color to the end of mark list, list rect shows in max size.
    mm           add color to mark list, the idx is which one cursor is in.
    dd           rmv color in mark list, the idx is which one cursor is in.
    
    Copy~
    reg "       reg +   (|quotequote|  |quote+|)
    Y/yy        C/cc     Copy HEX format 'FFFFFF'
    yr/yh/yl... cr/ch/cl

    see |colorv-formats| for details.

    Paste~
    <Ctrl-v>    paste color-text in register+ into ColorV.
    p/P         paste color-text in register" into ColorV.


                                                    *colorv-mapping-edit*
    Choose colors.~
    <2-Space>   <2-Leftmouse>   <Enter>
    <Enter>         Start editing RGB/HSV property chosen
    <Kenter>        Start editing RGB/HSV property chosen
    +=/-_           Tuning RGB/HSV property chosen
    <ScrollWheelUp>/<ScrollWheelDown> Tuning poperty(Only in X11 Gui)
    
    nn              Show colorname list
    na/ne           Entering a colorname.(W3C standard)   
    nx              Entering a colorname.(X11 standard)   
                    See |colorv-colorname| for details.


    ss              seach scheme of current Hex
    s1              show random scheme
    st              show top schemes
    sn              show newest schemes
    sh              show hotest schemes (Kuler only)
    sf              show Fav schemes
    sN              create new scheme


    ?               Show tips.
    H/<F1>          Show help document.
    
    q/Q             Quit ColorV.

    
    In Scheme Window~

    e               edit color under cursor
    f               Fav current scheme
    F               Unfav current scheme
    n               next scheme
    N/p             previous scheme
    sn              create new scheme
    K               search item under cursor with Kuler
    C               search item under cursor with colourlover
    q               quit scheme window

 6. COLORINFO                                  *colorv-colorinfo*
    
    This section contains information about colors.~
                                                     *colorv-colorname*
    The colorname are the the name of the colors, like 'red', 'green', 
    'blue', 'OrangeRed', 'cornsilk'.
    Note Exclude name in word with '_' or '-'.
         For example 'white-space' , 'black_sheep' will not be recongized.

    ColorV contains two types of names: X11 and W3C. 
    The X11 standard is used by vim and X Window system.
    The W3C standard is mostly used in HTML/CSS and browsers.
    They have few difference.
    
    There are several color names that have the same Hex.
        Aqua      Cyan            #00FFFF
        Magenta   Fuchsia         #FF00FF

    There are several clashes between X11 and W3C colornames.
        Gray    (X11)	#BEBEBE (W3C) 	#7F7F7F
        Green   (X11) 	#00FF00 (W3C) 	#007F00
        Maroon  (X11) 	#B03060 (W3C) 	#7F0000
        Purple  (X11) 	#A020F0 (W3C) 	#7F007F
    You can get the name table from
    http://en.wikipedia.org/wiki/X11_color_names
    
    Note    As the vim highlighting uses the X11 colornames.
            when |:ColorVEdit| in '*.vim' files, 
            ColorV will automatically edit it as X11 color names. 
            But the name showing in ColorV window is always W3C color name.

                                               *colorv-color-text*
                                                    *colorv-formats*
    The formats of color-text.~
    Predefined in ColorV.
    It is used when coping/pasting/viewing/... color-texts.

    There are following formats currently:

    The KEY means the abbrevation key used in mapping
    
    KEY  NAME    EXAMPLE                       DESCRIPTION
         HEX     FF00FF 334455
         HEX3    #CFF #F11
    #/s  HEX#    #FF00FF #00FFFF 
    0/x  HEX0    0xFFFF00   0xEE3399
    n    NAME    red/lime/blue                 (|colorv-colorname|)
    r    RGB     rgb(255,55,15)                (css1 standard)
    ar   RGBA    rgba(205,25,255,1.0)          (css2 standard)
    l    HSL     hsl(50,90%,40%)               (css3 standard)
    al   HSLA    hsla(230,30%,50%,1.0)         (css3 standard)
         glRGBA  glColor4f(1.00,0.5,1.00,1.00) (openGL color format)
    pr   RGBP    rgb(30%, 98%, 98%)             
    pa   RGBAP   rgba(100%,40%,100%,1.0) 
    h    HSV     hsv(360,100,100)
    m    CMYK    cmyk(25,41, 0,46) 

    e.g.: <leader>cim  will insert a CMYK color text

                                               *colorv-colors-generate*
    Methods for generating color list.~
    They are used for generating color lists.

    There are following type currently:

    The KEY means the abbrevation key used in mapping

    KEY  NAME                        DESCRIPTION
    h    Hue                         Hue 
    s    Saturation                  Saturation
    v    Value                       Value/Lightness
    m    Monochromatic               Generate by S and V s/v+{step}
    a    Analogous                   Generate colors h+15
    3    Triadic                     Generate 3 colors 
    4    Tetradic                    Generate 4 colors 
    n    Neutral                     Generate colors h+30
    c    Clash                       Generate 3 clash color 
    q    Square                      Generate 4 colors h+90
    5    Five-Tone                   Generate 5 colors 
    6    Six-Tone                    Generate 6 colors 
    2    Complementary               Generate opposite color h+180
    p    Split-Complementary         Generate 2 opposite colors
    l    Luma                        Generate by Luma+{step} ('yiq' only)
    
    g    Turn-To                     Generate colorlist by history_0 and history_1
                                     use |:ColorVTurn2| {hex1} {hex2}

  e.g.: <leader>cg5  will generate a Five-Tone list

                                               *colorv-color-space*
    Same S and V in HSV or HLS colors space don't result the same lumination.

    e.g:    Blue(0000FF) and Yellow(FFFF00) both '100' in S and V
            but their lumination differs obviously.

    The YIQ space (or other similar color space) can solve this.
    Their Y(luma) in YIQ colorspace are '89'(Yellow) and '11'(Blue)

    (See YIQ http://en.wikipedia.org/wiki/YIQ)
    
    Below is example table for HSV and YIQ space
    Use |:ColorVPreview| (<leader>cpp) to have an overview.

        HSV                YIQ (Blue)          YIQ (Yellow)
        #0000FF            #0000FF             #FFFF00 
        #4000FF            #2C00E9             #D3FF16
        #7F00FF            #5A00D8             #A5FF27
        #BF00FF            #8400C6             #7BFF39
        #FF00FF            #B000B2             #4FFF4D
        #FF00BF            #BA0076             #45FF89
        #FF007F            #C20041             #40FFBC
        #FF0040            #C90007             #36FFF8
        #FF0000            #D00000             #2FFFFF
        #FF4000            #AA0000             #55FFFF
        #FF7F00            #840400             #7BFBFF
        #FFBF00            #5F1F00             #A0E0FF
        #FFFF00            #393900             #C6C6FF
        #BFFF00            #0C4C00             #F3B3FF
        #7FFF00            #006000             #FF9FFF
        #40FF00            #007200             #FF8DFF
        #00FF00            #008500             #FF7AFF
        #00FF40            #007E00             #FF81FF
        #00FF7F            #007700             #FF87FF
        #00FFBF            #007031             #FF8FCE

 9. FUNCTIONS                                  *colorv-functions*

    Functions could be used~
    
    colorv#rgb2hsv({rgb})             Convert RGB to HSV.
                                      [r,g,b] 0~255 num
                                      returns [H,S,V] 0~360 0~100 num
                                      
    colorv#hsv2rgb({hsv})             Convert HSV to RGB.
                                      input [h,s,v] 0~360 0~100 num
                                      return [R,G,B] 0~255  num
                                      
    colorv#rgb2hex({rgb})             Convert RGB to HEX.
                                      input [r,g,b]  0~255  num/str
                                      return ffffff         str
                                      
    colorv#hex2rgb({hex})             Convert HEX to RGB.
                                      input ffffff          num/str
                                      return [r,g,b] 0~255  str
                                      
    colorv#hex2hsv({hex})             Convert HEX to HSV.
                                      input ffffff          num/str
                                      return [h,s,v] 0~360 0~100 num
                                      
    colorv#hsv2hex({hsv})             Convert HSV to HEX.
                                      input [h,s,v] 0~360 0~100 num
                                      return ffffff         str
                                      
    colorv#hex2hls({hex})             Convert HEX to HLS.
                                      input ffffff          num/str
                                      return [h,l,s] 0~360 0~100 num
                                      
    colorv#hls2hex({hsl})             Convert HLS to HEX.
                                      input [h,l,s] 0~360 0~100 num
                                      return ffffff         str

    colorv#hex2yiq({hex})           (y:0~100,i:-100~100,q:-100~100)
    colorv#yiq2hex({yiq})
    colorv#rgb2cmyk({rgb})          (c:0~100,m:0~100,y:0~100,k:0~100)
    colorv#cmyk2rgb({cmyk})

                                    hexa : 'ffffffff' hex with alpha channel.

                                    Add hexa1 and hexa2 with it's alpha 
                                    channel value.
